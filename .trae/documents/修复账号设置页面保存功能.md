# 账号设置页面保存功能修复计划

## 问题分析

通过检查代码，发现账号设置页面保存功能失败的可能原因是：

1. **数据字段不匹配**：在保存账号设置时，前端发送了所有表单字段，包括`avatar`，但后端可能不支持通过`/auth/me`端点更新头像
2. **头像处理机制**：头像上传功能是单独处理的，仅在本地更新`accountForm.value.avatar`，但在保存账号设置时，会将这个本地更新的头像数据（可能是base64字符串）发送到后端，导致后端拒绝请求

## 修复方案

1. **修改saveAccountSettings函数**：
   - 在调用`userStore.updateUserInfo`之前，创建表单数据的副本
   - 从副本中移除`avatar`字段，因为头像应该通过专门的上传接口处理
   - 仅将必要的字段（如email、phone、nickname、bio）发送到后端

2. **修改userStore.updateUserInfo方法**：
   - 添加数据验证，确保只发送后端支持的字段
   - 增强错误处理，提供更详细的错误信息

## 实现步骤

1. **修改SettingsView.vue中的saveAccountSettings函数**：
   ```javascript
   const saveAccountSettings = async () => {
     if (!accountFormRef.value) return
     
     try {
       await accountFormRef.value.validate()
       // 创建表单数据副本，移除avatar字段
       const formData = {
         ...accountForm.value
       }
       delete formData.avatar // 头像通过专门的上传接口处理，不包含在账号设置保存中
       // 调用store方法保存账号信息
       await userStore.updateUserInfo(formData)
       ElMessage.success('账号信息保存成功')
     } catch (error) {
       console.error('保存账号信息失败:', error)
       ElMessage.error(userStore.error || '保存失败，请稍后重试')
     }
   }
   ```

2. **增强userStore.updateUserInfo方法的错误处理**：
   ```javascript
   // 在updateUserInfo方法中添加更详细的错误日志
   console.error('更新用户信息失败:', error, '响应数据:', updateResult)
   ```

3. **验证修复效果**：
   - 运行构建命令确保代码无语法错误
   - 测试账号设置页面的保存功能，确认修改后的信息能正确保存到数据库

## 预期效果

修复后，账号设置页面的保存功能应该能够正常工作：
1. 用户修改的信息（如邮箱、手机号、昵称、个性签名）能够正确更新到数据库
2. 不会因为包含不支持的字段（如avatar）而导致保存失败
3. 在出现错误时，能够提供更详细的错误信息，便于调试

## 风险评估

- 低风险：仅修改前端代码，不影响后端逻辑
- 向后兼容：移除avatar字段不会影响其他功能，因为头像上传是单独处理的
- 可回滚：如果修复无效，可以快速恢复到原来的代码
